# Security Policy

## Reporting a vulnerability

If you discover a security issue in the **AetherCloud product**, please report it
privately. Do **not** open a public GitHub issue for security matters.

- **Email:** security@aethersystems.net
- Include: a description, reproduction steps, and impact. A proof-of-concept
  helps but is not required.
- Please give us a reasonable window to remediate before any public disclosure.

## Scope

- **In scope:** the AetherCloud desktop application and the aethersystems.net
  platform surfaces that serve it.
- **Out of scope:** this repository itself contains only marketing content and
  documentation — no application source code or secrets.

## Our commitment

AetherCloud is built around evidence-grade provenance: every file operation and
AI output is cryptographically signed (Protocol C) and appended to an immutable,
RFC 3161-timestamped chain of custody. We take reports seriously and will
acknowledge valid submissions.

Thank you for helping keep AetherCloud users safe.
